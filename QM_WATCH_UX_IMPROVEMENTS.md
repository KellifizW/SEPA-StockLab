# QM 盯盤模式 UX 優化總結

## 完成日期
2026-03-03

## 4 項主要改進

### 1️⃣ 動態評分右邊的 "-" 顯示改進
**問題:** 動態評分卡右邊當差異 < 1 時只顯示 "—"，看不到實際數據變化

**修復位置:** `templates/qm_analyze.html` 線 1481-1487
- **舊邏輯:** 只有 `|diff| >= 1` 才顯示，否則顯示 "—"
- **新邏輯:** 始終顯示實際差值（即使只有 +0.5 或 -0.3）

**效果:**
```
前: 動態評分 ← 符號右邊顯示 "—"
後: 動態評分 ← 符號右邊顯示 "+0.5" 或 "-0.3"
```

---

### 2️⃣ 評分明細 (Score Breakdown) 維度名稱化清晰
**問題:** 重複的 "ORH"、"ATR"、"NASDAQ" 等維度名稱不夠清楚，用戶看不懂 NASDAQ、Gap、HL 的含義

**修復位置:** `app.py` 線 3169-3318（w_breakdown 所有項目）

**改動清單:**
| 舊名稱 | 新名稱 | 說明 |
|--------|--------|------|
| ORH (x2) | ORH 突破 / ORH 失敗 | 區分上升和下降情況 |
| ATR | ATR 入場 | 加上「入場」說明用途 |
| NASDAQ | 市場環境 | 中文化，更直觀 |
| Gap | 開盤跳空 | 中文化，更具體 |
| HL | 低點結構 | 中文化，解釋什麼是低點結構 |
| 5m MA | 5分MA | 短期移動平均 |
| 1h MA | 60分MA | 中期移動平均 |
| Earnings | 財報期 | 中文化 |
| Breakout | 突破強度 | 更清晰的描述 |

**附加功能:** 評分明細現有 Tooltip
- HTML 中的 `data-bs-toggle="tooltip"` 使所有維度都可以悬停查看詳細說明
- 示例: "ORH 突破" → Tooltip: "三級全部突破上行，強烈看好"

### 3️⃣ 盤中訊號 (Intraday Commentary) 信號分類修復
**問題:** 信號列表顯示為 "5個看好 + 5個看淡"，但只列出 5 個，且無 emoji 標記難以區分

**根本原因:** 後端信號字串缺少 emoji 前綴，導致計數邏輯失敗
```python
# 舊代碼 (壞):
bullish = sum(1 for s in signals if "" in s)  # 空字符串，計數為 0

# 新代碼 (好):
bullish = sum(1 for s in signals if "🟢" in s)
bearish = sum(1 for s in signals if "🔴" in s)
neutral = sum(1 for s in signals if "⚠️" in s or "ℹ️" in s)
```

**修復位置:** `app.py` 線 2981-3161（所有 signals.append() 調用）

**信號分類規則:**
- **🟢 看好 (Bullish):** 
  - 突破 ORH (5分、30分)
  - 更高低點確認
  - 價格 > SMA20 或 > EMA65
  - 入場極佳/理想
  - 挑戰日高

- **🔴 看淡 (Bearish):**
  - 跌破 ORL
  - 跌破 SMA20 或 EMA65
  - 低點下移
  - 追價過高 (>1×ATR)
  - 財報黑色期
  - NASDAQ 停損區
  - 跳空過大

- **⚠️ 警告/注意 (Warning):**
  - 財報警告 (4-7天)
  - NASDAQ 警戒
  - 跳空注意
  - 入場謹慎

**效果:**
```
前: ⚖️ 平衡 (5個看好 vs 5個看淡) → [列出 5 個信號，無 emoji]
後: ⚖️ 平衡 (3個看好 vs 2個看淡，淨 +1) → [列出 5 個信號，每個有 emoji 標記]
```

### 4️⃣ ATR 入場評估卡顯示改進
**問題:** 卡片仍顯示「等待盤中 5 分鐘資料…」，使用者看不到 ATR 評估

**根本原因:** ATR 條件檢查錯誤
```python
# 舊代碼 (壞):
atr_daily = atr_daily or 0.0  # 如果 atr_daily=None，轉為 0.0
if current_price and atr_daily and lod:  # 0.0 是 Falsy，條件失敗！

# 新代碼 (好):
if current_price and atr_daily and atr_daily > 0 and lod:  # 明確檢查 > 0
```

**修復位置:** `app.py` 線 3046, 3052 與 `templates/qm_analyze.html` 線 1502-1504

**改動:**
1. **後端:** 加強 ATR 條件檢查（支援正確的 None/0 判斷）
2. **前端:** 優化 `renderAtrGauge()` 顯示邏輯，加入更清晰的追價分級說明

**ATR 追價分級顯示:**
| 距離 | 分類 | 色彩 | 建議 |
|------|------|------|------|
| < 0.4×ATR | 極佳 | 🟢 綠 | 理想買點 |
| 0.4-0.67×ATR | 理想 | 🔵 藍 | 可以買 |
| 0.67-1×ATR | 謹慎 | 🟡 黃 | 需確認 |
| > 1×ATR | 超高 | 🔴 紅 | 避免 |

## 使用者體驗改善

### Before (舊版)
```
動態評分  —         ← 看不到數值變化
── ORH    +3        ← 重複名稱
── ORH    —         ← 混淆
── ATR    0         ← 簡寫不明
── NASDAQ -1        ← 英文難懂
── El     0         ← 神秘の缩写
盤中訊號 ⚖️ 平衡 (5 vs 5)
── [5 個信號無 emoji] ← 無法分類
```

### After (新版)
```
動態評分  +0.5      ← 清楚看到分數上升
── ORH 突破  +3     ← 區分突破方向
── ORH 失敗  —      ← 無失敗信號
── ATR 入場  0      ← 清晰維度名稱
── 市場環境  -1     ← 中文化，可懸停看詳細說明
── 低點結構  0      ← 易懂
盤中訊號 ⚖️ 平衡 (3個看好 vs 2個看淡，淨 +1)
── 🟢 突破30分鐘高點... ← 一眼看出看好
── 🔴 跌破30分鐘低點... ← 一眼看出看淡
```
